# 向偶数舍入 (Round-To-Even)

笔记

  • 计算机内默认的舍入方式，也称为“向最接近值的舍入”
    • 其它方式会产生系统误差
  • 关键的设计决策是确定两个可能结果的中间数值的舍入
    • 确保舍入后的最低有效数字是偶数
      • 前提是最低有效数字右侧位串为
    • 比如: 向百分位舍入
      1.2349999    1.23   (Less than half way)
      1.2350001    1.24   (Greater than half way)
      1.2350000    1.24   (Half way一round up)
      1.2450000    1.24   (Half way一round dowmn)
  • 对于二进制而主
    • “Even” 意味着最低有效数字需要为
      • 前提是最低有效数字右侧的位串为
实例：二进制舍入到小数点后 2 位

# C 语言中的浮点数

笔记

  • float: 单精度浮点数
  • double: 双精度浮点数

类型转换

  • 当 int(32 位宽)，float 与 double 等类型之间进行转换时，基本原则如下：
    • double 或 float 转换为 int
      • 如果溢出或浮点数是 NaN，则转换结果没有意义
        • 通常置为 Tmin 或 Tmax
      • 否则，值会向零舍入
    • int 转换为 double
      • 能够精确转换
    • int 转换为 float
      • 不会溢出，但是可能被舍入(即精度受损)
